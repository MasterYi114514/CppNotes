# 1. 比特与字节

在计算机中，最小的数据单位是比特（bit），它可以表示两种状态：0 和 1。
一个字节（byte）通常由 8 个比特组成，可以表示 256（$2^8$）种不同的状态。

---------------------------------------------------------------------

比特（bit）：  0/1
字节（byte）： 0000 0000 (8个比特，每个位置上都可以是0或1)

---------------------------------------------------------------------

我们用小写的 b 来表示比特，用大写的 B 来表示字节。

```
b = bit = 二进制位
B = byte = 字节 = 8个比特
KB = 千字节 = 1024字节
MB = 百万字节 = 1024KB
GB = 十亿字节 = 1024MB
TB = 万亿字节 = 1024GB
Mbps(Mb/s) = 兆比特每秒
MBps(MB/s) = 兆字节每秒
```

内存是计算机中用于存储数据和程序的硬件设备，它由许多字节组成。内存的地址是用来标识每个字节位置的唯一标识符。

## 2.1 C++ 内存

在 C++ 中，内存可以分为以下几个部分：
 - **代码段：其中存储着 C++ 代码编译后生成的机器指令。**
	当程序运行时，CPU 会从代码段中读取指令并执行。
 - **静态数据区：也叫全局数据区。**
     - 只读数据区（.rodata）
     - 数据区（.data）
     - 未初始化数据区（.bss）
 - **动态数据区**
     - 堆区（Heap）
     - 栈区（Stack）

![[2.jpg]]

注：
1. 全局变量会存储在 `静态数据区` 中，根据是否初始化，分别存在 `未初始化数据区（.bss段）` 和 `数据区（包括.rodata段 和 .data段）` 中。
     - 更具体来说，由于 `.bss段` 中的值会被操作系统自动清零，因此我们会把“经过初始化但是初始化值为0”的数据也放到 `.bss段` 中。
     - 以上操作同时会导致所有的未初始化的全局变量和“经过初始化但是初始化值为0”的全局变量在程序开始运行时都是 0 。
2. 局部变量会存储在 `栈区` 中，函数调用时会在栈上分配空间来存储局部变量，函数返回时会释放这些空间。
     - 由于栈区不会被自动清零，因此局部变量在定义时如果没有被初始化，那么它的值就是一个随机值，可能是之前在该内存位置存储的数据残留。
     - MSVC 编译器在 `Debug` 模式下会把栈区的内存初始化为 `0xCC`，堆区的内存初始化为 `0xCD`，以帮助开发者更容易地发现未初始化变量的使用（这也导致了锟斤拷和烫烫烫的产生）。
3. 栈区是 CPU 高速运行的内存区域，我们要求更快的访问速度，整个栈几乎都在 L1/L2 缓存中，因此栈的区域很小，通常 Windows 系统默认的栈大小为 1MB，Linux 系统默认的栈大小为 8MB。

## 2.2 内存地址

**地址**： 内存中的每个字节都有一个唯一的地址，用于标识该字节的位置。地址是一个二进制数，通常以十六进制形式表示。 ^195831

地址的大小随着计算机体系结构的不同而不同。

注：
1. 对于 32 位系统，由于地址总线宽度为 32 位，因此我们也用 32 个二进制位来表示地址，通常写成十六进制也就是 `0x00000000` 到 `0xFFFFFFFF` 的形式。类似地，64位系统用 64 个二进制位来表示地址，通常写成十六进制也就是 `0x0000000000000000` 到 `0xFFFFFFFFFFFFFFFF` 的形式。
2. 不难计算，32 位地址能表示的地址空间为 $2^{32}$ 字节，即 4GB；64 位系统能表示的地址空间为 $2^{64}$ 字节，即 16EB（Exabyte）。后者的大小已经远超当前硬件所能支持的内存容量。
